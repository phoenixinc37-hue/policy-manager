# FIND IT — IMPLEMENTATION PLAN

## Goal
Move Find It from concept into an initial working web app prototype.

## Current Status
Product direction locked.

Completed:
- Concept brief
- MVP structure
- Core features
- Technical direction

## Build Objective
Produce an initial working prototype that allows a clinic to:
- search for an item
- see cabinet and shelf/drawer location
- browse by cabinet
- add/edit items as admin

## Construction Phases

### Phase 1 — Foundation
- app scaffold
- layout
- search-first home page
- global design system

### Phase 2 — Core Experience
- search results
- item detail
- cabinet browse
- mock data layer

### Phase 3 — Admin Experience
- add/edit item form
- category setup
- basic settings/setup guidance

### Phase 4 — Prototype Readiness
- realistic Rosslyn seed data
- polish
- mobile cleanup

## Immediate Construction Tasks
1. Create project/app folder
2. Initialize Next.js scaffold
3. Create search-focused landing/app shell
4. Build item result and detail pages
5. Build cabinet browse flow
6. Build admin add/edit item form
7. Seed with Rosslyn sample items

## Standard
The first version should feel fast, useful, and obvious — not bloated.

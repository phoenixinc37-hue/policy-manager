# codes/core-codes.md

## Question coding
- Kate questions must use **Q1-Q99**
- Scott questions use **Q100-Q199**
- Reuse the same Q number for the same unresolved question
- Do not assign a new number when the same question is still open
- If answering Scott's coded question, explicitly answer under that same Scott code

## Project coding
- Use `P*` plus short label for concurrent project separation
- Current examples:
  - `P*policy manager`
  - `P*PM`
  - `P* petes tire`
  - `P*cca letter`

## 7 Summits task coding
- Format: `[Summit].[Category].[Task].[Subtask](letter)`
- Preserve letters across versions when carrying work forward

## CSW coding
- Final approval questions in a CSW package use `CSW#`
- Scott replies with `CSW#.1 YES/NO`

## Filing/status codes
- `3001` = final sent message, filed for record, no follow-up needed
- `3002` = final sent message, active context/reference retained
- `3003` = final sent message, pending expected reply

## Operational shorthand
- `FTM` = File to Memory
- `SNS` = Start New Session
- `Dev-T` = Developed Traffic, avoid creating extra corrective traffic through sloppy or unverified output

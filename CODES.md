# CODES.md

Shorthand codes and session commands.

## Session / Memory Codes

- **FTM** — File to Memory. Move durable procedural info into `MEMORY.md` / FTM Log.
- **SM** — Specific Memory. Project-specific memory that must be organized and retained for the current build/project, but is not a permanent operational rule.
- **SNS** — Start New Session. Means prepare/move the foundational info needed for a clean new session.
  - If Scott uses SNS **without** a title, Kate’s first and only response should be: `title`
  - If Scott already includes the title in the SNS message, do **not** ask for title again.

## Question Codes

- **Kate question codes:** `Q1`–`Q100`
- **Scott question codes:** `Q101`–`Q200`
- Any sentence from Kate containing a question mark must use a `Q#` code.
- Keep questions to **1–2 per message**.

## Notes

- New sessions should be intentionally titled and indexed.
- Codes should stay short, clear, and reusable.

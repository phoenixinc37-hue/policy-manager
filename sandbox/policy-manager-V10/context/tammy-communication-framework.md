## Core Dynamic

Tammy's control mechanisms:
- **Narrative redirection** — shifts from concrete issue to abstract discussion (process, coordination)
- **Veto without alternative** — rejects plans, provides no replacement, system stalls
- **Status quo control** — delay → no plan → defaults to Tammy's preference
- **Non-response during planning** → rejection after proposal

Scott's shift: From seeking agreement → structured proposal-based execution

## 1/2/3 Communication Model

All communication with Tammy follows:
1. **Acknowledge** — brief, non-engaging
2. **Propose** — clear, concrete plan
3. **Require alternative + proceed if none**

Key rule: Objection without alternative = no impact
Key rule: Delay without alternative = ignored

## Decision Points System

Any date/event requiring assignment = a Decision Point.
Each must be:
1. Assigned to Tammy, OR
2. Assigned to Scott, OR
3. Explicitly shared

If not addressed → defaults to Scott's plan.

## 777 System (Message Refinement)

For tightening messages paragraph by paragraph:
- Numbers = current/final version
- Letters = previous version
- "keep 1/2/3" / "drop d" / "rewrite 2"

Purpose: eliminate drift, over-explaining, maintain precision.

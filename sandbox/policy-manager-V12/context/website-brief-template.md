# Website Brief Template

Use this at the start of any new website project.

## 1. Website name
- What is the project/site called?

## 2. Purpose
- What is the website supposed to do?
- Main goal: inform, generate leads, sell, book, explain, showcase, etc.

## 3. Audience
- Who is the website for?
- Primary visitor/customer/user?

## 4. Core offer
- What is being offered?
- Main products, services, or message?

## 5. Pages needed
- What pages should be included?
- Example: Home, About, Services, Pricing, Contact, FAQ

## 6. Must-have sections
- What sections absolutely need to appear?
- Example: hero, testimonials, CTA, pricing block, founder note, contact form

## 7. Style / vibe
- What should it feel like?
- Example: premium, modern, corporate, sharp, trustworthy, bold, simple

## 8. Content already available
- What content already exists?
- Copy, logos, photos, brand colors, pricing, PDFs, notes, screenshots

## 9. Features needed
- Any specific functionality?
- Example: form, booking, dashboard, member login, search, filters, upload, payments

## 10. Examples / references
- Any websites, screenshots, or styles to copy or loosely follow?

## 11. Technical / delivery needs
- GitHub repo?
- Live deployment?
- Mobile-first?
- Editable by Scott after stage 1?

## 12. Creation-stage definition of done
- What has to be true for Stage 1 to count as complete?
- Example: full site structure built, live link working, copy in place, mobile layout usable

## 13. Modification-stage priorities
- After Stage 1 is done, what kinds of detail changes are expected in Stage 2?
- Example: spacing, wording, colors, layout order, image swaps, button text
